## Git Push Blocker - Needs User Action
**Issue:** No valid GitHub PAT configured; `.git-credentials` file doesn't exist.

**Environment:**
- Repository: `https://github.com/alekssro/home-lab`
- Local branch: `feature/add-gemma4-e4b-model`
- Commit: `09fc99e`
- Remote URL configured: `https://github.com/alekssro/home-lab.git`
- `GITHUB_TOKEN` env var: empty/not set
- SSH client: not available

**Resolution Needed (User Action Required):**
1. **Generate PAT** at https://github.com/settings/tokens with:
   - Name: "home-lab-push"
   - Scopes: `repo` (full control of private repos) or `public_repo` + `workflow`
   - Expiration: Never or long duration (10 years)
   - Store securely
   
2. **OR set up SSH keys:**
   ```bash
   ssh-keygen -t ed25519 -C "alekssro@openclaw"
   cat ~/.ssh/id_ed25519.pub | ssh -t git@github.com "cat >> ~/.ssh/authorized_keys"
   chmod 600 ~/.ssh/id_ed25519
   ssh-copy-id git@github.com
   ```

3. **Once token is available**, set:
   ```bash
   export GITHUB_TOKEN="your-token-here"
   echo "https://x-access-token:${GITHUB_TOKEN}@github.com/" > ~/.git-credentials
   git config --global credential.helper store
   ```

4. Then retry: `git push origin feature/add-gemma4-e4b-model`
